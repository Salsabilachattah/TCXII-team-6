1 AUTHENTIFICATION&ACCÈS
1.1 Problème:“Email nonreconnu” ou“Motdepasseincorrect”
Symptôme:Erreur lors login.
Solutions :
1. Vérifiez email
• Compteavec email exact? (même casse compte)
• Exemple : alice@company.com̸ = Alice@Company.com
• Comptecréé avec Google/Microsoft? Utilisez même option
2. Réinitialisez mot de passe
• Page login → “Mot depasse oublié?”
• Email reçu : cliquez lien
• Créez nouveau mot de passe (min 8 char, majuscule, chiffre)
• Réessayez login
3. Compte pasactivé?
• Devriez recevoir email “Vérifiez votre email”
• Vérifiez spam/promotions
• Lien expiré (24h)? Cliquez “Renvoyer email”
4. Compte avec SSO?
• SicompanyutiliseSSO(AzureAD,Okta),allezloginSSO(pasemail/password)
• Contact IT company si lien SSO absent
Si rien marche : →support@doxa.dz (indiquez email, tentatives)
1.2 Problème:“Vous n’avez pas accès à ce projet/espace”
Symptôme:Page fermée, permission refusée.
Causes probables :
1. Pas membredel’espace
• Adminpasvousainvité
• Action : Demandez à admin : Paramètres → Membres →Inviter
2. Pas membreduprojet
• Adminvousainvité espace, mais pas ce projet
• Action : Admin →Projet →Paramètres → Ajouter vous
3. Mauvaise rolle
• Vous“Lecteur” mais besoin “Éditeur” (créer tâches)
• Action : Admin change votre rôle
4. Compte suspendu
• Paiement en retard? Espace passe “lecture seule”
• Action : Paramètres → Facturation → Payer facturation
5. URLvieille/cassée
• Lien projet ancien (projet supprimé/archivé)
• Action : Allez accueil → cherchez projet
Rapide test :
• Allez accueil (doxa.dz/dashboard)
• Voyez vous AUCUNprojet/espace?
• Si oui →contact admin
1.3 Problème:“2FAbloqué” ou“Coded’authentification invalide”
Symptôme:Nepeutpasentrer code 2FA.
Solutions :
1. Code incorrect?
• TOTPcodes changent chaque 30 secondes
• Entrez rapidement (pas plus de 30s après génération)
• Ré-essayez nouveau code
2. Horloge synchronisation
• Votre téléphone horloge décalée = code invalide
• Solution : Synchronisez l’heure (Paramètres téléphone → Date & Heure)
• Puis regénérez code
3. Utilisez code de secours
• Avezcodes de secours (10 codes, reçus activation 2FA)?
• Utilisez un code (single-use)
• Gardez remaining codes safe
4. 2FA cassé/Authenticator app perdu
• Si perdu téléphone = problème!
• Contact support@doxa.dz
• Vérification identité requise (email account, autre proof)
Prévention :
• Sauvegardez codes de secours safe (password manager, papier)
• Testez 2FA après activation (logout/login test)
2 PROJETS&TÂCHES
2.1 Problème:“Je n’arrive pas à créer/modifier tâches”
Symptôme:Boutons grisés, “Accès refusé”, ou tâches pas sauvegardées.
Solutions :
1. Rôle insuffisant
• Vous“Lecteur”? Besoin “Éditeur”
• Contactez admin pour upgrade rôle
2. Projet archivé
• Projet archivé = lecture seule automatiquement
• Action : Admin →Paramètres projet → Déarchiver
3. Espace suspendu (paiement retard)
• Espace dépasse plan limit?
• Exemples : Starter max 5 projets, 5 membres
• Action : Upgrade plan ou archivez tâches/projets
4. Navigateur problème
• Cache vieux, JavaScript désactivé
• Solutions :
• Vider cache (Ctrl+Shift+Suppr) → site Doxa seul
• Reload page (Ctrl+F5)
• Essayer autre navigateur (Chrome, Firefox, Edge)
5. Offline
• Pasinternet connection?
• Doxanécessite connexion (pas offline mode)
Diagnostic :
1. Essayez créer tâche test simple (1 mot, pas attachments)
2. Si échoue → navigateur/connexion problem
3. Si succès mais avant non → data tâche (trop grande, char spécial?)
2.2 Problème : “Mes tâches disparues” ou “Tâches archivées non vi
sibles”
Symptôme:Tâches existaient, plus là.
Solutions :
1. Filtres appliqués

3. Suppression accidentelle
• Tâche supprimée (pas archivée)
• Doxanegardepashistorique suppression
• Impossible récupérer (design purpose)
4. Mauvaise vue/projet
• Vousregardez bon projet? Bon workspace?
• Créé dans “Projet A” mais cherchez “Projet B”?
• Utilisez recherche globale (Ctrl+K) : “Find task X”
5. Permissions changées
• Adminachangépermissions → tâches cachées
• Demandezadmin
Test :
• Créez nouvelle tâche test → elle s’affiche?
• Si oui : problème vue/filtre
• Si non: problème espace/permissions

3 COLLABORATION&COMMENTAIRES
3.1 Problème:“Messages/commentaires ne sauvegardent pas”
Symptôme:Écrivez commentaire, cliquez envoyer, disparaît.
Solutions :
1. Connexion perdue
• Réseau dropped pendant envoi
• Solution : Vérifiez internet (wifi, data)
• Rééssayez commentaire
2. Session expirée
• Onglet ouvert longtemps sans activité (> 30min inactif)
• Logout automatique
• Solution : Reconnectez-vous
• Recopier texte si long
3. Caractères spéciaux problématique
• Emoji, accents extrêmes, Unicode rare?
• Solution : Simplifiez texte (pas emoji pour test)
4. Limite de caractère
• Commentaire max5000 caractères
• Plus = rejet silencieux
• Solution : Divisez en 2 commentaires
Prévention :
• Écrivez commentaires COURTS (< 500 char idéal)
• Copier avant envoyer (Ctrl+C) pour safety
• Logout et reconnectez 1x/jour (refresh session)
3.2 Problème:“Les @mentionsnemarchentpas”
Symptôme:@Alice mais elle pas reçu notification.
Solutions :
1. Mauvais nom?
• Tapez “@Ali” → autocomplete? Sélectionnez bon nom
• Pasd’autocomplete → person pas dans espace/projet
• Assurez alice est membre projet
2. Notifications mutées
• Alice a désactivé notifications
• Allez Profil → Préférences → “Notifications” → réactivez
3. Notification spam / email
• Alice a peut-être reçu, mais email spam
• Vérifiez spam/promotions
4. Connectez avant d’écrire
• Besoin être connecté pour @mention
• Connectez-vous d’abord
Rapide fix :
• Utilisez format : “@alice” (pas “@Alice”)
• Sélectionnez depuis autocomplete (pas tapez)
• DemandezAlice vérifier notifications

3. GitHub integration pas connectée

• Autorisez Doxa access
4. Repository pas dans authorized repos list
• Connectez GitHub → sélectionnez repos
• Doit inclure votre repo
• Si pas là : reconnectez + sélectionnez
5. Branch/PR before integration connected?
• Si PRcréée AVANTGitHub intégration setup
• Peut pas retroactively lier
• NewPRswithcloses #task-XYZ = linkés ok
Test :
1. Créez NEWPR
2. Description : “Closes #task-ABC123”
3. Submit PR
4. Refresh Doxa task page
5. Voyez PR lié en section “Linked”?
5 PERFORMANCE&BUGS
5.1 Problème:“Doxatrès lent” ou “Page freeze”
Symptôme:Tableau chargement lent, interaction gelée.
Solutions :
1. Trop detâches?
• Tableau avec 1000+ tâches = lent
• Solution : Filtrez/archivez anciennes tâches
• Laissez max 200-300 visibles
2. Trop decolonnes custom?
• Si 50+ colonnes = UI lourde
• Solution : Supprimez colonnes non-utilisées
3. Cache navigateur plein
• Ctrl+Shift+Suppr → Clear browsing data
• Sélectionnez “Cached images” + “Cookies”
• Juste Doxa.dz
• Reload
4. Navigateur/RAM limitation
• Beaucoup onglets ouverts?
• Fermez autres, redémarrez navigateur
• Ouutilisez mode incognito (RAM fresh)
5. Internet lent
• Vitesse connexion < 1 Mbps?
• Testez speedtest.net
• Utilisez téléphone hotspot (plus rapide?)
6. Pièces jointes massives
• Tâche avec 50 fichiers 100MB?
• Solution : Archivez vieux fichiers, ou stockez cloud autre
• Doxapasideal pour huge files
Performance check :
1. Ouvrez DevTools (F12)
2. Onglet “Network” → mesure page load
3. Doit être < 2 sec
4. Si > 5sec →contact support
5.2 Problème:“Erreur ’Something went wrong”’ ou erreur 500
Symptôme:Message erreur générique, ou page blanche.
Solutions :
1. Reload page
• F5(simple refresh)
• Ctrl+F5 (hard refresh, vider cache)
• Ctrl+Shift+Suppr (clear cookies + cache)
2. Logout/Login
• Paramètres → Logout
• Reconnectez-vous
3. Essayer autre navigateur
• Chrome→Firefox
• Oumodeincognito (pas extensions)
4. Vérifier status Doxa
• Allez status.doxa.dz
• Est-ce qu’incident reporté?
• Si oui →waitfor fix
5. Clear browser extensions
• Ad-blockers, password managers peuvent interfér
• Désactivez temporairement
• Retry
6. Contact support
• Si persiste : support@doxa.dz
• Envoyez : screenshot error, time/timezone, navigateur/version
• Include console error (F12 → Console onglet → copy messages)
6 DONNÉES&EXPORTS
6.1 Problème:“Impossible exporter données”
Symptôme:Bouton export grisé ou erreur.
Solutions :
1. Rôle insuffisant
• Besoin “Éditeur” ou mieux pour exporter
• “Lecteur” pas peut exporter (security)
• Contact admin pour upgrade rôle
2. Trop detâches
• Export >10,000 tâches = peut timeout
• Solution : Filtrez d’abord (par date, catégorie)
• Exportez en chunks
3. Format pas compatible
• Besoin CSV mais avez JSON?
• Solution : Utilisez autre format, import outil autre
• Oudemandezformatautre
4. Timeout/erreur réseau
• Export prend du temps (> 10s parfois)
• Internet connection dropped?
• Solution : Réessayez, ou contact support
Alternative :
• APIexport : Utilisez API REST pour export custom
• Documentation : api.doxa.dz/export
6.2 Problème:“Impossible importer données (CSV)”
Symptôme:Upload CSV, erreur ou données pas importées.
Solutions :
1. Format CSVincorrect
• Headers must match Doxa colnames (title, assignee, due_date, etc.)
• Encoding : UTF-8 required (pas ANSI, pas Excel default)
• Solution : Ouvrez CSV en Notepad++ → Encoding UTF-8 → Save
2. Données validation
• Dates format correct? (YYYY-MM-DD, pas DD/MM/YYYY)
• Emails valides?
• Assignees existent dans espace?
• Solution : Validez données avant import
3. File too large
• Maxupload50MB
• Divisez en 2 fichiers si plus grand
4. Espace full
• Dépassez limite plan (ex : Starter max 1000 tâches)?
• Supprimez/archivez vieilles avant import
Import example :
title,description,assignee_email,due_date,priority
"Fix login bug","Authentication broken",alice@company.com,2025-01-20,high
"Add dark mode","New feature request",bob@company.com,2025-02-01,normal
7 ACCOUNT&FACTURATION
7.1 Problème:“Facturation confuse” ou “Charge incorrecte”
Symptôme:Montant inattendu, ou confusion plan.
Solutions :
1. Vérifiez plan actuel
• Paramètres → “Facturation” → “Plan actuel”
• Vuprix? Vous sur Starter (29€), Pro (79€), ou Enterprise?
2. Changement plan mid-month
• Si montez plan mid-month : crédit prorata
• Exemple : 15€ utilisé, +50€ upgrade = facture ∼35€ prochain mois
• Normal!
3. Annual vs Monthly confusion
• Payez annuel (débit 1x/an) vs mensuel (1x/mois)?
• Vérifiez Facturation : quelle periode?
4. Pas payement détecté
• Carte rejetée? Essayez autre méthode (PayPal, SEPA)
• Expiré? Mettez à jour carte
• Contact billing@doxa.dz si question
5. Veux remboursement?
• 30jours satisfaction garantie
• Contact support@doxa.dz
• Preuve + raison = remboursement
7.2 Problème:“Espace suspendu” ou “Accès lecture-seule”
Symptôme:Nepouvezpascréer tâches, message espace “lecture-seule”.
Solutions :
1. Paiement en retard
• Facturation non payée (7+ jours)?
• Espace passe automatiquement “lecture-seule”
• Solution : Paramètres → Facturation → Payer
• Après paiement confirmé →réactivé
2. Dépassez limite plan
• Starter max 5 membres? Invité 6e?
• Espace pause →upgrade plan
• Solution : Starter→Pro (79€), ou supprimez membre
3. Violation ToS
• Abusdétecté (spam, malware, abuse)?
• AdminDoxapeutsuspendre
• Solution : Vérifiez email violation notice
• Répondez appeal si légitime (false positive)
4. Limite tâches/stockage
• Starter max 1000 tâches, 5 GB stockage?
• Solution : Archivez tâches, supprimez attachments volumineux
